❌ Issue: White Text on Light Yellow (~1.5:1 Contrast)

The announcement banner uses color: #ffffff on background: #ffff99. The contrast ratio is approximately 1.5:1 — WCAG AA requires 4.5:1 for normal text, 3:1 for large text. ❌ Issue: Status Conveyed by Color Alone

Service status (green = OK, red = degraded/outage, amber = investigating) is communicated only through color. Users who are colorblind or who view the page in grayscale cannot distinguish between these states. WCAG 1.4.1 requires that color not be the only visual means of conveying information. Fix: add a text label or icon (●, ▲, ✗) alongside the color, or use distinct background patterns/shapes.

❌ Issue: Links Indistinguishable from Body Text

The links in the paragraph above use the same color: #0f172a as surrounding text and have no underline. Users who don't use color to identify links (colorblind users, or users on older displays) cannot tell which words are clickable. WCAG 1.4.1 requires links to be visually distinct from surrounding text without relying on color alone. Fix: add text-decoration: underline, or use a sufficiently contrasting link color with a hover/focus underline.

Quick Search

❌ Issue: Placeholder-Only Label + Low-Contrast Placeholder

The search field has no <label> — it relies solely on the placeholder "Search services, logs, or users." Placeholders disappear on input and have insufficient contrast (#cccccc on white = ~1.6:1). Separately, the Search button's :focus outline uses #dddddd, which is virtually invisible. Fix: add a visible or visually-hidden <label>, use placeholder contrast ≥ 4.5:1, and make the focus ring clearly visible.

❌ Issue: Chart Uses Color as the Only Differentiator

The bar chart above uses five different colors to represent different services. There are no axis labels, no text values on the bars, no legend, and no title or aria-label on the chart container. A colorblind user or someone using a screen reader receives no information at all. Fix: add visible axis labels and a legend, place data values on or near bars, and wrap the chart in an element with role="img" aria-label="Bar chart showing API Gateway traffic is highest at 145 requests…" or provide a data table alternative.
