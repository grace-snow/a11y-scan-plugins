Personal Information

All fields are required unless noted.

❌ Issue: Input Has No Label

The "First name" field has no <label> element. The placeholder disappears as soon as the user starts typing and is never read by all screen readers. Fix: add <label for="first-name">First name</label> linked to the input via id="first-name".

❌ Issue: Label Not Programmatically Associated

The label "Last name" has no for attribute, and the input has no id. Visually it looks correct, but screen readers cannot pair them. Clicking the label also won't focus the input. Fix: add for="last-name" to the label and id="last-name" to the input.

Contact Details

❌ Issue: Missing autocomplete Attribute

The email field uses type="text" instead of type="email" and has no autocomplete attribute. WCAG 1.3.5 requires autocomplete on inputs that collect personal data so browsers and assistive technologies can auto-fill them. Fix: add type="email" autocomplete="email".

❌ Issue: Required Field Not Indicated

This phone number field is required for form submission but shows no visual or programmatic indication of that requirement. Users who skip it will only find out after submitting. Fix: add a visible asterisk and/or the word "required," plus required and aria-required="true" on the input.

❌ Issue: Select Has No Label

This dropdown has no associated <label>. When a screen reader focuses it, all it can announce is the first selected option ("Select your country"). Users don't know what kind of information they're selecting. Fix: add a linked <label for="country">Country</label> and id="country" on the select.

Your Message

❌ Issue: Textarea Has No Label

The message textarea relies only on a placeholder for identification. Placeholders are not a replacement for labels: they disappear on input, have lower contrast, and are not consistently announced by all assistive technologies. Fix: add <label for="message">Message</label>.

Zip / Postal Code

This field is required.

❌ Issue: Error Message Not Associated with Input

The "This field is required" error message appears above the input but has no programmatic link to it. Screen readers won't announce the error when the user focuses the field. Fix: add aria-describedby="zip-error" to the input and id="zip-error" to the error message, plus role="alert" or aria-live="polite" on the error.

I'm interested in (select all that apply)

❌ Issue: Checkbox Group Missing fieldset / legend

These related checkboxes are not wrapped in a <fieldset> with a <legend>. Screen readers need the group context to understand that all checkboxes relate to "I'm interested in." Without it, each checkbox sounds independent. Fix: wrap with <fieldset><legend>I'm interested in</legend>…</fieldset>.

❌ Issue: Button Has No Accessible Name

The blue "Submit" button above has no text content, no aria-label, and no title. Screen readers will announce it as simply "button" with no context about what it does. Fix: add visible text ("Submit"), or aria-label="Submit registration form".

❌ Issue: Submit Input Has Empty Value

This <input type="submit"> has value="", making the button appear blank. Screen readers announce the value as the button's name — an empty value leaves users without any description of what the button does. Fix: set a descriptive value such as value="Create Account".
