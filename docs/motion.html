❌ Issue: Infinite Animation Without prefers-reduced-motion Check

The spinning loader above plays indefinitely and has no @media (prefers-reduced-motion: reduce) override. For users with vestibular disorders or motion sensitivities, persistent spinning animations can cause nausea or disorientation. Fix: add @media (prefers-reduced-motion: reduce) { .spinner { animation: none; } } or replace the animation with a static indicator for those users.

❌ Issue: Animated Element Without prefers-reduced-motion Check

The gradient bar above pulses continuously at approximately 0.3 Hz. While this rate does not reach the WCAG 2.3.1 seizure threshold (3 Hz), it still lacks a @media (prefers-reduced-motion: reduce) override. Fix: add @media (prefers-reduced-motion: reduce) { .slow-pulse { animation: none; } } to respect the user's system motion preference.

❌ Issue: Auto-Scrolling Content With No Pause, Stop, or Hide Control

The news ticker scrolls automatically and continuously. WCAG 2.2.2 (Pause, Stop, Hide) requires that any auto-moving, auto-scrolling, or auto-updating content that lasts more than 5 seconds must have a mechanism to pause, stop, or hide it. There is no such control here. Users with cognitive disabilities, attention disorders, or those who use screen magnification find persistent motion highly disruptive. Fix: add a "Pause" button that stops the animation, using animation-play-state: paused.

❌ Issue: Video Autoplays, Has No User Controls, and Has No Captions

This video has three separate WCAG failures:
1. Autoplay (WCAG 1.4.2): The video starts playing automatically with no user interaction. Users with cognitive disabilities or who use screen readers may be confused or overwhelmed by unexpected audio/video.
2. No controls (WCAG 2.1.1): The controls attribute is missing, so keyboard users have no way to pause, stop, or adjust the video.
3. No captions (WCAG 1.2.2): There is no <track kind="captions"> element. Deaf and hard-of-hearing users cannot follow any audio content.
Fix: remove autoplay, add controls, and provide a WebVTT captions file via <track kind="captions" src="captions.vtt" srclang="en" label="English" default>.
