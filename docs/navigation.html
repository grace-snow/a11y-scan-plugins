❌ Issue: Focus Outline Removed

The CSS rule :focus { outline: none; } removes the visible focus indicator for all interactive elements sitewide. Keyboard and switch-access users rely on the focus outline to know where they are on a page. Removing it is a WCAG 2.4.7 (Focus Visible) failure. Fix: never remove focus outlines globally — instead, style them to match your design using :focus-visible.

❌ Issue: Skip Link Always Hidden

A "Skip to main content" link exists in the HTML but uses position: absolute; top: -999px without a matching :focus style to bring it into view. Keyboard users who Tab to it can't see or use it, defeating its purpose. Fix: add .skip-link:focus { top: 0; } so the link appears when focused.

This "button" is a <div> with an onclick handler. It is not reachable by Tab key, cannot be activated with Enter/Space, and announces nothing meaningful to screen readers. Fix: use a real <button> element, or add tabindex="0", role="button", and keyboard event handlers.

❌ Issue: Positive tabindex Disrupts Tab Order

Using tabindex="1", tabindex="3", and tabindex="5" forces these links to be reached before any other focusable element on the page, and in a non-visual order. This breaks the natural, predictable Tab flow. Fix: remove all positive tabindex values. Only use tabindex="0" (to make non-interactive elements focusable) or tabindex="-1" (to manage focus programmatically).

❌ Issue: Dialog Has No Focus Management or Accessible Name

This element has role="dialog" but: (1) focus is never moved into it when it "opens," (2) it has no aria-label or aria-labelledby to announce its title, and (3) focus can escape freely without being trapped inside. Fix: use a real <dialog> element, set focus to the first interactive element inside it on open, add aria-labelledby pointing to the dialog title, and implement a focus trap.

❌ Issue: No <main> Landmark

This page's primary content is wrapped in a <div> instead of a <main> element. Screen reader users can jump directly to main content using landmark shortcuts, but only if the page uses <main>. Fix: replace the outer <div class="page-wrapper"> with <main class="page-wrapper">.
